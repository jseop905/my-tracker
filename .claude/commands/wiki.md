name: wiki
description: Update or add project wiki documentation in docs/wiki/
---

`docs/wiki/` 문서를 갱신하거나 새로 추가한다. `.claude/skills/wiki-management.md` 규칙을 따른다.

## 실행 모드

### 1. 대상 지정 갱신 (인자 있음)

사용자가 대상 문서와 변경 사항을 지정한 경우:

1. 대상 문서를 읽는다
2. 관련 코드의 현재 상태를 확인한다
3. 코드 기준으로 문서를 수정한다
4. frontmatter `updated` 날짜를 갱신한다
5. 변경 내용을 사용자에게 요약 보고한다

예시:
```
/wiki modules.md 결제 모듈 추가 반영
/wiki conventions.md 에러 핸들링 패턴이 Result 타입으로 변경됨
/wiki data-model.md Order 테이블 추가
```

### 2. 대화형 갱신 (인자 없음)

인자 없이 실행한 경우:

1. `docs/wiki/`의 기존 문서 목록과 `updated` 날짜를 확인한다
2. 문서가 없으면 `/project`를 먼저 실행하라고 안내한다
3. 문서가 있으면 목록을 보여주고 질문한다:
   - "어떤 문서를 갱신할까요? 또는 어떤 변경이 있었는지 알려주시면 관련 문서를 찾겠습니다."

### 3. 동기화 확인 (`sync`)

`/wiki sync`로 실행한 경우:

1. 모든 wiki 문서의 `scope`와 `updated` 날짜를 읽는다
2. 각 문서의 scope에 해당하는 코드의 최근 변경 이력을 확인한다 (git log)
3. 문서보다 코드가 최신인 경우 해당 문서를 "갱신 필요"로 표시한다
4. 결과를 테이블로 보고한다
5. 갱신이 필요한 문서가 있으면 어떤 것을 갱신할지 사용자에게 확인한다

자동 갱신하지 않는다. 보고만 하고 사용자가 판단한다.

## 갱신 시 규칙

1. 기존 내용을 함부로 삭제하지 않는다 — 특히 사용자가 검증한 내용
2. `[추정]` 태그가 있던 항목이 코드로 확인되면 태그를 제거한다
3. 새로 추가하는 불확실한 정보에는 `[추정]` 태그를 붙인다
4. 한 문서의 변경이 다른 문서에 영향을 주는 경우 함께 갱신한다
   (예: 모듈 추가 시 modules.md + architecture.md)
